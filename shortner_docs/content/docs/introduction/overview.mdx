title: API Overview
description: High-level overview of the WarpIt API.
---

The WarpIt API allows you to programmatically warp URLs, manage custom codes, and retrieve original links. Our API is RESTful and uses JSON for both requests and responses.

## Base URL

All API requests should be made to the following base URL:

```bash
http://localhost:4000/api/v1
```

## Authentication

Most endpoints require a Bearer token in the `Authorization` header. See the [Authentication](/introduction/authentication) guide for details.

## Response Format

All responses follow a standard format:

```json
{
  "success": true,
  "data": { ... },
  "message": "Success message",
  "status": 200
}
```

## Error Handling

Standard HTTP status codes are used to indicate the success or failure of an API request.

| Status | Code | Description |
| :--- | :--- | :--- |
| 200 | OK | The request was successful. |
| 400 | Bad Request | The request was invalid or missing parameters. |
| 401 | Unauthorized | Authentication failed or was not provided. |
| 404 | Not Found | The requested resource was not found. |
| 409 | Conflict | The custom code is already taken. |
| 500 | Internal Server Error | Something went wrong on our end. |
